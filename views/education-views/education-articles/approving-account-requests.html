Approving Account Requests

Keep student and faculty access up to date by reviewing, approving, or declining account requests from the Users page Account Requests table.

Open the Users tab

  1. From the Dashboard, expand the Admin menu on the left.
  2. Select Users to see everyone tied to the programs you administer.
  3. Use the filter buttons to focus on Pending account requests or narrow by campus/program.

Review each request

Click a name to open the side panel and confirm the individualâ€™s campus, role (student, faculty, or staff), and the program they selected. If anything looks off, leave an internal note or reach out before approving.

  • Documents: Verify any attachments, such as faculty verification forms.
  • History: Check whether the person previously had an account that was deactivated.

Approve, decline, or request edits

  1. Use the action buttons at the bottom of the panel.
  2. Approve immediately activates the account and sends the welcome email.
  3. Decline archives the request and lets you add a note explaining why.
  4. Needs Edits (if enabled) sends the request back to the applicant with your instructions.

You can always reopen the Users tab later to reactivate an archived account or adjust program access if a student changes cohorts.

Keep the queue clean

  • Approve daily so students can move on to registration and compliance tasks.
  • Archive duplicate or withdrawn requests to avoid confusion.
  • Document any unusual situations in the notes field for your fellow coordinators.
